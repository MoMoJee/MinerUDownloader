# MinerU Downloader 升级方案 v2.0

## 一、错误处理升级

### 1.1 新增 `errors.py`

集中定义全部错误码、分类及提示，不在业务代码中硬编码 code 字符串。

```
ErrorCategory（枚举）
  AUTH        — Token 无效 / 过期
  RATE_LIMIT  — 每日配额耗尽（-60018 / -60019）
  TRANSIENT   — 暂时性故障，可重试（-10001 / -60001 / -60007 / -60009 …）
  FILE_ERROR  — 文件本身问题，不重试（-60002 ~ -60006 / -60015 ~ -60017 / -60020 ~ -60022）
  CONFIG      — 调用参数/格式错误（-500 / -10002 / -60008 / -60011 / -60012 / -60013）
  UNKNOWN     — 兜底

ERROR_CATALOG: dict[str, ErrorInfo]
  key   = code 字符串（"-60018"、"A0202" …）
  value = ErrorInfo(message, hint, category)

class MinerUApiError(Exception)
  .code: str
  .info: ErrorInfo

parse_api_response(data: dict) -> None
  — 若 data["code"] != 0，查 catalog 构造 MinerUApiError 并抛出
```

### 1.2 `api.py` 改动

- `apply_upload_urls` / `poll_batch` 中所有 `raise RuntimeError(...)` 改为调用
  `parse_api_response(data)` 抛出 `MinerUApiError`
- 上层（tui.py / cli.py 的处理线程）捕获 `MinerUApiError`，按 category 分支处理：
  - `AUTH / RATE_LIMIT` → 调用 `token_manager.report_error(idx, category)`，自动切换 Token 重试
  - `TRANSIENT` → 沿用现有重试逻辑
  - `FILE_ERROR` → 标记该文件 FAILED，跳过重试
  - `CONFIG` → 直接记录日志并通知用户

## 二、负载均衡升级

### 2.1 Token 输入格式（多 Token）

| 来源 | 格式 |
|---|---|
| `mineru_config.yaml` | `token: "sk-a"` 或 `token: ["sk-a", "sk-b"]` |
| `--token` 参数 | 逗号分隔字符串 `"sk-a,sk-b"` |
| TUI / CLI 输入 | 逗号分隔字符串 |
| `MINERU_TOKEN` 环境变量 | 逗号分隔字符串 |

**优先级（高→低，完全替换，不合并）：**
```
TUI/CLI 输入 > --token 参数 > mineru_config.yaml > MINERU_TOKEN 环境变量
```

### 2.2 新增 `token_manager.py`

```
TokenState（枚举）: ACTIVE / RATE_LIMITED / INVALID

@dataclass TokenEntry
  .token: str
  .state: TokenState
  .enabled: bool      # 用户在 TUI 手动禁用
  .used_batches: int  # 已分配批次数（统计用）

class TokenManager
  __init__(tokens: list[str])
  .entries: list[TokenEntry]           # 有序列表

  get_token() -> tuple[int, str]
    — 轮询取下一个 state==ACTIVE 且 enabled==True 的 Token
    — 若全部不可用，抛出 NoAvailableTokenError

  report_error(idx: int, category: ErrorCategory) -> None
    — AUTH / RATE_LIMIT → 修改对应 entry.state
    — TRANSIENT → 不修改（交给重试）

  toggle_enabled(idx: int) -> None
    — 翻转 entry.enabled

  has_available() -> bool
  status_lines() -> list[str]          # 供 TUI / CLI 展示用

  @staticmethod
  parse_tokens(raw: str | list) -> list[str]
    — 字符串按逗号分割；列表直接用；去空格去重
```

### 2.3 `config.py` 改动

- `_DEFAULTS["token"]` 类型保持 `""` 或 `list[str]`，YAML 两种格式均支持
- `load_config` 不再强制转为字符串，返回原始值（str 或 list）
- `merge_cli_args` 中 `--token` 值存为逗号分隔字符串，不拆分（由 TokenManager 负责解析）
- `save_config` 保存时：单 Token 存字符串，多 Token 存列表

### 2.4 TUI 改动（SelectScreen）

**选项面板 Token 区域改造：**
```
[Token(s)，逗号分隔多个]
┌─────────────────────────────┐
│ sk-aaa, sk-bbb              │  ← Input#input-token
└─────────────────────────────┘
负载均衡: ☑ 启用              ← Checkbox#cb-lb（有多 Token 时自动勾选）

Token 状态列表（仅多 Token 时显示）：
  ☑ [0] sk-aa…aaa   ACTIVE    0 批
  ☑ [1] sk-bb…bbb   ACTIVE    0 批
  （T 键在文件树聚焦时切换文件，在此列表聚焦时切换 Token 启用状态）
```

**组件：**
- `ListView` id=`token-list`，每项格式见上
- `Checkbox` id=`cb-lb`，控制是否启用负载均衡
- `on_input_changed("#input-token")` → 解析多 Token，刷新列表
- T 键绑定：`action_toggle_item` 内部判断 focused 组件类型

**ProgressScreen 右侧新增 Token 状态区：**
```
Token 状态:
  [0] sk-aa…  ✓ ACTIVE   12 批
  [1] sk-bb…  ✗ RATE_LIMITED
```

### 2.5 API 调用层改动

`submit_and_upload_batch` / `apply_upload_urls` 签名改为接受 `token_manager: TokenManager`
而非单一 `token: str`，每次申请 URL 前调用 `token_manager.get_token()`，
捕获 `MinerUApiError(AUTH/RATE_LIMIT)` 后自动 `report_error` 并重试（最多重试所有可用 Token 一轮）。

`poll_batch` 同理：每次轮询请求前用相同 idx 的 Token（batch 创建时绑定），
若该 Token 在轮询期间失效则切换并记录日志。

### 2.6 CLI 改动（`_phase_options`）

Token 输入提示改为支持多个：
```
Token(s)（逗号分隔多个，回车保持）:
```
若检测到多个 Token，提示：
```
检测到 2 个 Token，已启用负载均衡模式 (y/n)? [y]
```

## 三、实施顺序

1. `errors.py` — 独立新文件，无依赖
2. `token_manager.py` — 依赖 errors.py
3. `config.py` — 多 Token 存储格式
4. `api.py` — 接入 errors.py + TokenManager
5. `tui.py` — Token 列表 UI + ProgressScreen Token 状态
6. `cli.py` — 多 Token 输入提示
7. `main.py` — Token 优先级逻辑 + 构建 TokenManager
